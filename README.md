## Возможности

- **Живые данные** — текущее число активных коннектов обновляется каждые 30 секунд
- **KPI-карточки** — сейчас активно, пик за период, среднее в час, всего точек собрано
- **График по часам** — история подключений с переключением периода: 1 день / 3 дня / 7 дней
- **Диаграмма по времени суток** — доля активности по четырём слотам (ночь / утро / день / вечер)
- **Тепловая карта** — день недели × час, среднее за 7 дней
- **Топ пиковых часов** — рейтинг самых нагруженных часов
- **Светлая и тёмная темы** — переключение в один клик
- **Ротация лога** — автоматически хранит данные за последние 30 дней (~43 200 точек)

## Требования

| Компонент | Версия |
|---|---|
| Docker | 20.10+ |
| Docker Compose | v2 (плагин) или v1 |
| Запущенный MTProto-контейнер | с именем `mtproto` |
| Доступ root | для cron и `/var/log` |

> Скрипт ожидает, что MTProto-прокси работает в Docker-контейнере с именем **`mtproto`** и слушает порт **443**. Если у вас другое имя контейнера или порт — отредактируйте переменные `CONTAINER_NAME` и `PORT` в начале `setup.sh` перед запуском.

## Установка

```bash
sudo bash setup.sh
```

Скрипт автоматически:

1. Проверяет наличие Docker и контейнера `mtproto`
2. Создаёт директорию `/opt/mtproto-monitor/`
3. Генерирует все необходимые файлы (см. [Структура файлов](#структура-файлов))
4. Регистрирует cron-задачу для сбора данных каждую минуту
5. Собирает Docker-образ и запускает сервис через `docker compose`
6. Проверяет доступность API

После успешной установки в консоли будет выведен адрес дашборда и полезные команды.

### Как работает сбор данных

Раз в минуту cron запускает `collect.sh`, который выполняет:

```bash
docker exec mtproto ss -tn | grep ":443" | wc -l
```

Результат записывается в `/var/log/mtproto_conns.log` в формате:

```
<unix_timestamp>,<count>
```

Пример строки:
```
1744132931,495
```

Лог автоматически ротируется: хранятся только последние 43 200 строк (~30 дней при сборе раз в минуту).

## API

Все эндпоинты возвращают JSON.

### `GET /api/current`

Возвращает текущее число активных подключений (в реальном времени, не из лога).

```json
{
  "timestamp": 1744132931,
  "connections": 495
}
```

---

### `GET /api/stats?days=7`

Возвращает агрегированную статистику за `days` дней (по умолчанию 7).

**Параметры:**

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `days` | int | `7` | Число дней глубины выборки |

**Ответ:**

```json
{
  "hourly": {
    "2026-04-01 14:00": 452,
    "2026-04-01 15:00": 498
  },
  "heatmap": {
    "Пн": [120, 95, 80, 70, 65, 72, 150, 340, 480, 520, 510, 490,
           460, 430, 470, 500, 520, 560, 590, 610, 620, 600, 540, 300],
    "Вт": [...]
  },
  "kpi": {
    "current": 495,
    "avg_per_hour": 452,
    "max": 743,
    "max_ts": 1744121460,
    "total_points": 1433,
    "days": 7
  }
}
```

---

### `GET /api/history?hours=24`

Возвращает сырые точки лога за последние `hours` часов.

**Параметры:**

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `hours` | int | `24` | Глубина выборки в часах |

**Ответ:**

```json
{
  "data": [
    {"ts": 1744046531, "count": 412},
    {"ts": 1744046591, "count": 418}
  ]
}
```

## Управление

### Просмотр логов бэкенда

```bash
docker logs mtproto-monitor -f
```

### Просмотр сырых данных коннектов

```bash
tail -f /var/log/mtproto_conns.log
```

### Перезапуск сервиса

```bash
cd /opt/mtproto-monitor && docker compose restart
```

### Остановка сервиса

```bash
cd /opt/mtproto-monitor && docker compose down
```

### Ручной запуск сборщика данных

```bash
/opt/mtproto-monitor/collect.sh
```

### Проверка cron-задачи

```bash
crontab -l | grep mtproto
```

После установки в `/opt/mtproto-monitor/` будут находиться:

```
/opt/mtproto-monitor/
├── collect.sh          # Cron-скрипт сбора коннектов (каждую минуту)
├── main.py             # FastAPI-бэкенд
├── Dockerfile          # Образ для mtproto-monitor

    └── index.html      # Фронтенд дашборда

/var/log/mtproto_conns.log     # Лог данных (unix_ts,count)
/var/log/mtproto_conns_cron.log # Лог cron-задачи
```

## FAQ

**Q: Контейнер называется не `mtproto`, что делать?**

Перед запуском `setup.sh` открой файл и измени переменную `CONTAINER_NAME` в начале скрипта:
```bash
CONTAINER_NAME="мой-контейнер"
```

**Q: MTProto слушает не 443, а другой порт?**

Аналогично измени переменную `PORT`:
```bash
PORT=8888
```

**Q: Дашборд показывает «Нет связи»?**

Убедись, что контейнер `mtproto-monitor` запущен:
```bash
docker ps | grep mtproto-monitor
```

Если контейнер не запущен, подними его:
```bash
cd /opt/mtproto-monitor && docker compose up -d
```

**Q: Данных нет, хотя прокси работает?**

Cron собирает данные раз в минуту. После установки подожди 1–2 минуты и обнови страницу. Также можно запустить сборщик вручную:
```bash
/opt/mtproto-monitor/collect.sh && tail -1 /var/log/mtproto_conns.log
```

**Q: Как сменить порт дашборда?**

Измени переменную `SERVICE_PORT` в `setup.sh` перед установкой:
```bash
SERVICE_PORT=9090
```
